# Meeting Assistant — 设计文档

> 日期：2026-04-03
> 状态：待实施

## 1. 概述

一个实时会议助手桌面应用，帮助用户在飞书远程会议中获得战略性发言建议。应用通过捕获系统音频进行本地语音识别，结合 LLM 分析会议内容，实时提供会议摘要和发言建议，帮助用户在技术评审、Review 会等场景中展示领导力。

## 2. 核心需求

- 实时捕获远程会议音频（包括远程参会者和自己的声音）
- 本地语音识别，中英文混合支持
- 基于可配置模板的 LLM 发言建议
- 延迟目标：≈ 7-10 秒
- 完全本地化运行（Whisper 本地 + Ollama），隐私优先

## 3. 技术栈

| 层 | 技术 | 说明 |
|---|------|------|
| 框架 | Tauri v2 | 桌面应用框架 |
| 前端 | React + TypeScript + Tailwind CSS | UI 层 |
| 音频捕获 | cpal (Rust) | 跨平台音频 I/O |
| 语音识别 | whisper-rs | whisper.cpp 的 Rust 绑定 |
| LLM 调用 | reqwest (Rust) | HTTP 客户端，调用 OpenAI 兼容 API |
| 存储 | serde_json + rusqlite | 配置用 JSON，会议记录用 SQLite |

## 5. 音频管线

### 5.1 音频捕获

- 应用启动时枚举系统音频设备，用户选择：
  - **输入设备**（麦克风）— 采集自己的声音
  - **输出捕获设备**（BlackHole）— 采集远程参会者的声音
- 两路音频混合为单声道 16kHz PCM（Whisper 要求的格式）
- 音频写入环形缓冲区（Ring Buffer），每 5 秒切一段送给 Whisper

### 5.2 BlackHole 设置

- BlackHole 是免费开源的 macOS 虚拟音频驱动
- 用户需在 macOS Audio MIDI Setup 中创建"多输出设备"（Multi-Output Device），包含耳机 + BlackHole
- 系统输出设为多输出设备，这样声音同时发给耳机（正常听）和 BlackHole（给 Whisper 录）
- 应用首次启动时提供引导步骤

### 5.3 Whisper 处理

- 模型：`ggml-small.bin`（约 466MB，首次启动自动下载）
- 存放路径：`~/.meeting-assistant/models/`
- 语言设置：自动检测（支持中英混合）
- 每 5 秒取一段音频送入推理，M2 上推理耗时约 1-2 秒
- 端到端延迟：5 秒窗口 + 1-2 秒推理 ≈ 7 秒

## 6. LLM 发言建议引擎

### 6.1 会议模板系统

每个模板定义：

```yaml
name: "技术评审会"
description: "架构设计、方案评审等技术讨论"
system_prompt: |
  你是一个高级技术领导的发言顾问。
  你的目标是帮用户展示技术深度和领导力。
  建议的发言角度：风险把控、架构方向、资源协调、技术选型。
trigger_hints:
  - "大家觉得怎么样"
  - "有没有问题"
  - "这个方案"
advice_style: "leadership"
```

### 6.2 预置模板

| 模板 | 用途 | 发言风格 |
|------|------|---------|
| 技术评审会 | 架构设计、方案评审 | 展示领导力、把控方向 |
| Review 会 | 代码/方案评审 | 一针见血的问题、发现薄弱点 |
| 项目同步会 | 进度对齐、风险同步 | 识别风险、推动 action item |
| 头脑风暴 | 创意发散 | 提出创意、构建他人想法 |

用户可编辑预置模板，也可创建自定义模板。

### 6.3 触发机制

不是每句转录都调用 LLM，按策略触发：

1. **定时摘要** — 每 30 秒将最近转录文本发给 LLM，更新会议摘要
2. **发言时机检测** — 基于规则检测：
   - 有人提了问题但无人回答（沉默 > 3 秒）
   - 话题切换（关键词变化明显）
   - 有人在做总结或征求意见（匹配 trigger_hints）
   - 技术讨论中出现可质疑的点
3. **规则命中后调 LLM** — 发送上下文 + 模板 system_prompt，生成具体建议

### 6.4 LLM Provider 配置

- 支持 OpenAI 兼容 API 格式（`base_url` + `api_key` + `model`）
- 默认走本地 Ollama（`http://localhost:11434/v1`）
- 用户可在设置中切换到 OpenAI、Claude 等

## 7. 前端 UI

### 7.1 布局

三区布局，窄长窗口，放在屏幕侧边：

```

├──────────────────────────────────────────┤
│                                          │
│  会议摘要                                 │
│  • 要点 1                                │
│  • 要点 2                                │

│                                          │
├──────────────────────────────────────────┤
│                                          │
│  发言建议                       ● 新建议!  │
```

### 7.2 交互特性

- **顶部控制栏**：当前模板名称（可点击切换）、会议计时、设置/暂停/停止按钮
- **会议摘要面板**：实时滚动更新的要点列表，每 30 秒刷新，始终显示当前焦点话题
- **发言建议面板**：新建议时高亮闪烁提醒，显示发言理由 + 具体建议 + 角度，旧建议自动淡出
- **窗口特性**：默认窄长、支持 always-on-top 置顶、支持调整大小

### 7.3 UI 设计流程

用户使用 Pencil 工具出 UI 原型图，开发者根据原型用 React + Tailwind 实现。

## 8. 数据流与存储

### 8.1 运行时数据流（全在内存）

```
音频块(5s) → Whisper转录 → TranscriptStore追加
                                   │
                          ┌────────┴────────┐
                          ▼                 ▼
                    定时摘要(30s)      规则检测触发
                          │                 │
                          ▼                 ▼
                    LLM生成摘要       LLM生成建议
                          │                 │
                     emit事件           emit事件
                          │                 │
                          ▼                 ▼
                    前端摘要面板       前端建议面板
```

### 8.2 持久化存储

| 数据 | 存储方式 | 路径 |
|------|---------|------|
| 会议模板 | JSON 文件 | `~/.meeting-assistant/templates/` |
| LLM 配置 | JSON 文件 | `~/.meeting-assistant/config.json` |
| 音频设备偏好 | JSON 文件 | 同上 |
| 会议记录（可选） | SQLite | `~/.meeting-assistant/history.db` |

### 8.3 会议记录（可选）

- 会议结束时弹窗询问"是否保存本次会议记录？"
- 保存内容：转录文本、摘要、发言建议列表、会议时长
- 可在"历史"页面回顾

## 9. 隐私设计

- 所有音频数据只在内存中处理，不落盘
- Whisper 本地运行，音频不上传
- 如果用本地 Ollama，全程离线
- 如果用云端 LLM，只发送转录文本（不发音频）

## 10. 明确不做（v1 范围外）

- **Speaker Diarization（说话人识别）** — v1 不区分发言者，后续版本考虑
- **飞书 API 集成** — 飞书无公开实时转录 API，不走此路线
- **Windows/Linux 支持** — v1 仅支持 macOS（BlackHole 依赖）

## 11. 首次使用引导

1. 检测是否安装 BlackHole，未安装则提示下载链接
2. 引导在 macOS Audio MIDI Setup 创建多输出设备
3. 首次启动自动下载 Whisper `ggml-small.bin` 模型（显示进度）
4. 提供"测试音频"按钮，确认音频捕获正常
5. 配置 LLM Provider（默认 Ollama）
